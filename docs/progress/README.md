# Progress Records

本目录用于记录项目的月度进展摘要。

进展记录不替代任务卡。任务卡记录单个任务的完整过程，progress 记录一段时间内项目整体向哪里走。

## Monthly Template

```md
# Progress: YYYY-MM

## Completed

...

## System Changes

...

## Decisions

...

## Learning Notes

...

## Next Priorities

...
```

## What To Record

适合记录：

1. 本月完成的任务。
2. 系统结构变化。
3. 重要 ADR。
4. 反复出现的问题。
5. 下个月优先事项。

不适合记录：

1. 每个小 commit 的细节。
2. 未验证的实现计划。
3. 可以直接从任务卡读取的完整过程。

## Principle: Why Progress Records Exist

长期项目会产生很多任务卡、commit 和讨论。progress 文件给你一个高层回顾入口。

它回答的是：

```text
这个项目最近实际前进到了哪里？
```

而不是：

```text
每一行代码为什么变了？
```
